# Configuring the Client

In this chapter, you will configure the Prisma client to connect to your server. We cover both the GUI app and the CLI configuration.

## What You Need

Before starting, make sure you have:

- Your **server's IP address** (e.g., `203.0.113.45`)
- Your **Client ID** (from `prisma gen-key` on the server)
- Your **Auth Secret** (from `prisma gen-key` on the server)
- The **port** your server listens on (default: `8443`)

:::tip
If you used `--setup` when installing the server, these values are saved in `/etc/prisma/.prisma-credentials` on your server. View them with:
```bash
cat /etc/prisma/.prisma-credentials
```
:::

## Option 1: prisma-gui (Desktop App)

### Creating a Profile

1. **Open prisma-gui**
2. Click **"New Profile"** or the **+** button
3. Fill in the following fields:

| Field | What to enter | Example |
|-------|--------------|---------|
| **Profile Name** | A name for this connection (anything you want) | `My Server` |
| **Server Address** | Your server's IP address and port | `203.0.113.45:8443` |
| **Client ID** | The Client ID from gen-key | `a1b2c3d4-e5f6-7890-...` |
| **Auth Secret** | The Auth Secret from gen-key | `4f8a2b1c9d3e7f6a...` |
| **Transport** | How to connect (start with QUIC) | `QUIC` |
| **Cipher Suite** | Encryption algorithm | `ChaCha20-Poly1305` |

4. Click **Save**

### Transport settings

For a basic QUIC connection (recommended for beginners):

- **Transport:** QUIC
- **Skip Certificate Verify:** Turn ON (since we are using a self-signed certificate)

For TCP connection (if UDP is blocked):

- **Transport:** TCP
- Leave other settings as default

### Connect

1. Select your profile from the list
2. Click **Connect**
3. The status indicator should turn green when connected

## Option 2: CLI Configuration

### Write the client config

Create a file called `client.toml` on your local computer.

**On Linux/macOS:**

```bash
nano ~/client.toml
```

**On Windows (PowerShell):**

```powershell
notepad $env:USERPROFILE\client.toml
```

Paste the following configuration. **Replace the placeholder values with your own:**

```toml title="client.toml"
# ============================================================
# Prisma Client Configuration
# ============================================================

# ── Local Proxy Addresses ─────────────────────────────────────
# These are the addresses on YOUR computer where Prisma listens
# for connections from your browser and other apps.

# SOCKS5 proxy address. Your browser will connect here.
# "127.0.0.1" means only this computer can use it.
# ":1080" is the port number.
socks5_listen_addr = "127.0.0.1:1080"

# HTTP proxy address (optional). Some apps prefer HTTP proxy.
# Remove this line if you don't need it.
http_listen_addr = "127.0.0.1:8080"

# ── Server Connection ─────────────────────────────────────────
# The address of your Prisma server (the VPS you set up).
# Replace with your server's actual IP address and port.
server_addr = "YOUR-SERVER-IP:8443"

# Encryption algorithm. ChaCha20 is fast on all devices.
# Alternative: "aes-256-gcm" (faster on CPUs with AES hardware)
cipher_suite = "chacha20-poly1305"

# Transport method. "quic" is fastest and recommended.
# If QUIC doesn't work (UDP blocked), change to "tcp".
transport = "quic"

# Set to true when using self-signed certificates (from gen-cert).
# Set to false when using Let's Encrypt or other trusted certificates.
skip_cert_verify = true

# ── Identity ──────────────────────────────────────────────────
# These MUST match the values in your server's config.
# Generated by "prisma gen-key" on the server.
[identity]
client_id = "YOUR-CLIENT-ID"          # Paste your Client ID here
auth_secret = "YOUR-AUTH-SECRET"      # Paste your Auth Secret here

# ── Logging ───────────────────────────────────────────────────
[logging]
level = "info"      # "debug" for troubleshooting, "info" for normal use
format = "pretty"   # Human-readable output
```

### Replace the placeholders

You **must** replace three values:

1. `YOUR-SERVER-IP` -- Your VPS IP address (e.g., `203.0.113.45`)
2. `YOUR-CLIENT-ID` -- The Client ID from `prisma gen-key`
3. `YOUR-AUTH-SECRET` -- The Auth Secret from `prisma gen-key`

### Example completed config

Here is what a completed config looks like (with fake values):

```toml title="client.toml (example with filled values)"
socks5_listen_addr = "127.0.0.1:1080"
http_listen_addr = "127.0.0.1:8080"
server_addr = "203.0.113.45:8443"
cipher_suite = "chacha20-poly1305"
transport = "quic"
skip_cert_verify = true

[identity]
client_id = "a1b2c3d4-e5f6-7890-abcd-ef1234567890"
auth_secret = "4f8a2b1c9d3e7f6a0b5c8d2e1f4a7b3c9d0e6f2a8b4c1d7e3f9a5b0c6d2e8f"

[logging]
level = "info"
format = "pretty"
```

### Validate the config

```bash
prisma validate -c ~/client.toml
```

Expected output:
```
Configuration is valid.
```

## Choosing a Transport

Here is a quick reference for when to use each transport:

| Situation | Transport | Config value |
|-----------|-----------|-------------|
| Normal network, best speed | QUIC | `transport = "quic"` |
| UDP is blocked | TCP | `transport = "tcp"` |
| Need to hide server IP (with CDN) | WebSocket | `transport = "ws"` |
| Heavy censorship (with CDN) | XPorta | `transport = "xporta"` |

For beginners, **start with QUIC**. It is the fastest and works on most networks. If it does not work, switch to TCP.

## Setting Up System Proxy

After Prisma is running, you need to tell your computer to send traffic through it. There are two approaches:

### Option A: Configure your browser only

This sends only your browser's traffic through Prisma. Other apps connect directly.

**Firefox:**
1. Open Settings > Network Settings
2. Select "Manual proxy configuration"
3. Set SOCKS Host: `127.0.0.1`, Port: `1080`
4. Select "SOCKS v5"
5. Check "Proxy DNS when using SOCKS v5"

**Chrome/Edge:**
Chrome uses your system proxy settings. See Option B below, or use a proxy extension like SwitchyOmega:
1. Install the SwitchyOmega extension
2. Create a new profile of type "Proxy Profile"
3. Set Protocol: SOCKS5, Server: `127.0.0.1`, Port: `1080`
4. Click the extension icon and select your profile

### Option B: Configure system-wide proxy

This sends ALL your computer's traffic through Prisma.

**Windows:**
1. Open Settings > Network & Internet > Proxy
2. Under "Manual proxy setup," click "Set up"
3. Turn on "Use a proxy server"
4. Address: `127.0.0.1`, Port: `8080`
5. Click Save

**macOS:**
1. Open System Settings > Network
2. Select your active connection (Wi-Fi or Ethernet)
3. Click "Details" > "Proxies"
4. Enable "SOCKS Proxy"
5. Server: `127.0.0.1`, Port: `1080`

**Linux (GNOME):**
1. Open Settings > Network > Network Proxy
2. Select "Manual"
3. Socks Host: `127.0.0.1`, Port: `1080`

### Option C: TUN mode (advanced)

TUN mode creates a virtual network device that captures ALL traffic automatically. No per-app or system proxy configuration needed. This is covered in [Going Further](./advanced-setup.md).

## Testing from the Command Line

You can test your proxy connection without configuring your browser using `curl`:

**SOCKS5 test:**

```bash
curl --socks5 127.0.0.1:1080 https://httpbin.org/ip
```

**HTTP proxy test:**

```bash
curl --proxy http://127.0.0.1:8080 https://httpbin.org/ip
```

If Prisma is working, you should see your **server's IP address** in the response, not your own.
