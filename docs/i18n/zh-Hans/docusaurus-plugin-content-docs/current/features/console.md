# 控制台

Prisma 控制台是用于监控和管理代理服务器的实时 Web 界面。它使用 Next.js 16、shadcn/ui、Recharts 和 TanStack Query 构建为静态站点，由 Prisma 服务器直接提供服务。

## 前提条件

- 一个已启用[管理 API](/docs/features/management-api)的运行中的 Prisma 服务器
- 控制台静态文件（预构建或从源代码构建）

## 设置

### 使用预构建文件

从[最新版本](https://github.com/Yamimega/prisma/releases/latest)下载 `prisma-console.tar.gz` 并解压：

```bash
mkdir -p /opt/prisma/console
tar -xzf prisma-console.tar.gz -C /opt/prisma/console
```

### 从源代码构建

```bash
cd apps/prisma-console
npm ci
npm run build
```

静态文件输出到 `apps/prisma-console/out/`。

### 服务端配置

在 `server.toml` 中将服务器指向控制台文件：

```toml
[management_api]
enabled = true
listen_addr = "0.0.0.0:9090"
auth_token = "your-secure-token-here"
console_dir = "/opt/prisma/console"  # 或 "./apps/prisma-console/out"
```

启动服务器后访问 `https://your-server:9090/` 即可打开控制台。

### 使用 CLI（`prisma console`）

`prisma console` 命令可自动下载并启动控制台，无需手动设置。

#### 基本用法

```bash
prisma console --mgmt-url https://127.0.0.1:9090 --token your-secure-token
```

该命令会从 GitHub Releases 下载最新控制台并缓存到本地，启动本地服务器将 API 请求代理到管理 API。桌面系统会自动打开浏览器。

#### 所有参数

| 参数 | 默认值 | 描述 |
|------|--------|------|
| `--mgmt-url` | 自动检测 | 管理 API URL（省略时从 `server.toml` 自动检测） |
| `--token` | -- | 认证的 Bearer 令牌（省略时从 `server.toml` 读取 `management_api.auth_token`） |
| `--config` / `-c` | `./server.toml` | 用于自动检测 `--mgmt-url` 和 `--token` 的 `server.toml` 路径 |
| `--listen` / `-l` | `127.0.0.1:9091` | 控制台服务器的本地地址 |
| `--no-open` | `false` | 不自动打开浏览器 |
| `--daemon` / `-d` | `false` | 作为守护进程在后台运行 |
| `--console-dir` | `~/.prisma/console` | 缓存控制台静态文件的路径 |

#### 从 server.toml 自动检测

当省略 `--mgmt-url` 和 `--token` 时，CLI 会读取 `server.toml`（从当前目录或 `--config` 指定的路径）并自动提取 `management_api.listen_addr` 和 `management_api.auth_token`：

```bash
# 从当前目录的 server.toml 自动检测所有配置
prisma console

# 从指定配置文件自动检测
prisma console -c /etc/prisma/server.toml
```

#### 守护进程模式

在后台运行控制台服务器：

```bash
# 以守护进程启动
prisma console -d

# 检查守护进程状态
prisma console status

# 停止守护进程
prisma console stop
```

守护进程将 PID 写入 `~/.prisma/console.pid`，日志写入 `~/.prisma/console.log`。

#### 架构：静态文件服务 + 反向代理

通过 `prisma console` 启动时，CLI 启动一个轻量级 HTTP 服务器，它：

1. **提供静态文件** -- 来自缓存目录的预构建控制台 SPA
2. **反向代理 API 请求** -- 所有 `/api/*` 和 `/api/ws/*` 请求转发到管理 API URL
3. **注入认证** -- `--token` 自动添加到代理请求中

```
浏览器 → prisma console（本地 :9091）→ 静态文件（控制台 SPA）
                                      → /api/* → 代理 → prisma-server:9090
                                      → /api/ws/* → WebSocket 代理 → prisma-server:9090
```

这允许在不配置 CORS 和不直接暴露管理 API 的情况下访问控制台。

## 认证

控制台使用基于令牌的认证。在登录页面输入服务器配置中的 `management_api.auth_token`。令牌存储在浏览器的会话存储中，并以 `Bearer` 令牌的形式随每个 API 请求发送。

所有 `/console/*` 路由都受保护——未认证用户将被重定向到 `/login`。

## 架构

控制台构建为静态单页应用程序（SPA），由 Prisma 服务器的管理 API（axum）提供服务。生产环境无需单独的 Node.js 进程。

```
浏览器 → prisma-server:9090 → 静态文件（控制台）
                              → /api/*（REST + WebSocket）
```

控制台的 API 调用直接发送到同源管理 API 端点。WebSocket 连接使用 `?token=` 查询参数进行认证（因为浏览器 WebSocket API 无法发送自定义头部）。

## 页面

### 概览

主控制台页面显示：
- **指标卡片** -- 活跃连接数、总上传/下载字节数、运行时间
- **迷你趋势图** -- 每个指标卡片内嵌迷你折线图，显示 5 分钟趋势
- **服务器健康评分** -- 环形仪表盘（0-100），基于 CPU、内存、证书到期、错误率和连接负载综合计算
- **流量图表** -- 实时字节/秒，支持时间范围选择（实时/1H/6H/24H/7D）和 Mbps 切换。实时模式显示最近 60 秒的数据。
- **传输类型饼图** -- 按传输类型分组的连接分布
- **连接时长直方图** -- 连接持续时间分布
- **GeoIP 饼图** -- 活跃连接的国家分布（需要在服务端路由配置中设置 `geoip_path`；未配置时显示占位提示）
- **实时连接地图** -- SVG 世界地图，按 GeoIP 国家显示活跃连接，圆点大小与连接数成正比
- **连接表格** -- 活跃连接的对端 IP（已去除端口号）、传输类型、模式、字节计数和断开按钮。表格上方显示汇总统计卡片（活跃连接数、总上传、总下载）。默认按 IP 分组显示。

数据源：WebSocket 推送（每秒指标）+ REST 轮询（每 5 秒连接状态）。

### 速度测试

通过代理连接运行吞吐量测试：
- **测试服务器** -- 4 个服务器：Cloudflare（25 MB）、Cloudflare（100 MB）、Hetzner、OVH
- **测试运行器** -- 多流下载（4 个并行流）和上传速度测量
- **实时进度** -- 测试期间实时显示下载/上传 Mbps 和进度条
- **结果** -- 完成后以卡片形式显示下载速度、上传速度和延迟
- **趋势图** -- 显示历史测试记录中下载/上传走势的折线图（2 条或以上记录时显示）
- **扩展统计** -- 所有历史记录中下载和上传的最小值、最大值和中位数
- **历史记录** -- 持久化测试历史（最多 50 条）存储在 localStorage 中，以列表形式显示并带相对时间
- 支持中途启动和停止测试

### 服务器

服务器信息：
- **健康状态** -- 服务器状态标签、版本和运行时间，每 10 秒自动刷新
- **配置** -- 监听地址、QUIC 监听地址、协议版本、DNS 上游、最大连接数、超时时间、日志级别/格式、路由规则数
- **功能标签** -- 端口转发、伪装、CDN 和端口跳跃状态（开/关）
- **TLS 信息** -- TLS 启用/禁用状态、证书和密钥文件路径
- **端口转发表格** -- 活跃的端口转发，显示端口、协议、绑定地址、活跃连接数和流量计数。支持完整的 CRUD 操作：直接在控制台中创建、编辑和删除端口转发。

### 系统

系统监控：
- **系统卡片** -- 版本、平台、PID、CPU 和内存使用率仪表
- **实时资源图表** -- 双轴面积图（左轴 CPU %，右轴内存 %），60 点滚动窗口，每 2 秒更新一次，图表下方显示当前值
- **证书到期** -- 倒计时，颜色编码（绿色 &gt;30 天、黄色 7-30 天、红色 &lt;7 天）
- **活跃监听** -- 所有监听地址和协议的表格

### 连接事件

实时 WebSocket 推送的连接/断开事件：
- **虚拟化事件列表** -- 高性能滚动，支持数千条事件
- **类型筛选** -- 按事件类型筛选：全部、连接、断开、错误
- **搜索** -- 跨事件详情（对端地址、传输方式、客户端名称）全文搜索
- **自动滚动** -- 自动跟随新事件；用户向上滚动时暂停
- **颜色编码标签** -- 绿色表示连接、红色表示断开、黄色表示错误事件

数据源：WebSocket 推送（`/api/ws/connections`）。

### 客户端

客户端管理：
- **客户端列表** -- 显示所有授权客户端的名称、状态（启用/禁用），点击可进入详情页
- **客户端标签与权限** -- 为客户端添加标签，可按标签筛选列表。在详情页管理每客户端权限（端口转发、UDP、目标地址、端口、最大连接数、带宽）。
- **客户端分享对话框** -- 生成即用型客户端配置，支持三种格式：TOML 配置（带语法高亮）、`prisma://` URI 或二维码。可从客户端列表或详情页访问。
- **客户端详情** -- 每客户端带宽限制（可编辑）、配额利用率进度条、流量图表、过滤后的连接表格
- **连接历史** -- 每客户端时间线视图，显示 24 小时内的连接事件和流量峰值
- **批量导入/导出** -- 将所有客户端导出为 JSON，从文件导入客户端
- **添加客户端** -- 生成新的 UUID + 认证密钥对，密钥仅显示一次
- **编辑客户端** -- 更新名称、切换启用/禁用、配置带宽/配额限制
- **删除客户端** -- 从认证存储中移除客户端

更改立即生效——无需重启服务器。

### 路由

可视化路由规则编辑器：
- **规则列表** -- 所有规则按优先级排序，显示条件、操作和启用状态
- **内联编辑** -- 点击任何规则字段（条件、值、操作）即可直接在表格中编辑，无需打开对话框。更改通过管理 API 立即保存。
- **规则编辑器** -- 用于创建新规则的对话框表单，包含条件类型、值和操作
- **扩展的规则类型** -- 支持 DOMAIN、DOMAIN-SUFFIX、DOMAIN-KEYWORD、IP-CIDR、GEOIP、PORT 和 ALL 规则类型，带有自动补全建议
- **规则预设** -- 4 个类别（隐私、网络、区域、兜底）共 8 个预设模板，一键应用
- **批量删除** -- 选择多条规则并批量删除
- **切换/删除** -- 内联启用、禁用或删除规则

详见[路由规则](/docs/features/routing-rules)了解规则类型。

### 设置

带选项卡的服务器配置编辑器：
- **通用** -- 监听地址（可编辑）、DNS 上游、连接超时、端口转发（含端口范围、允许/拒绝端口、IP 白名单）、管理 API 开关、自动备份间隔
- **伪装与 CDN** -- ALPN、Salamander 密码、H3 掩护、CDN 路径、XPorta 配置（会话路径、编码、超时、会话数、Cookie）
- **流量与性能** -- 流量整形、拥塞控制、端口跳跃、DNS、反 RTT 设置
- **TLS 与安全** -- 证书信息、仅传输层加密、协议版本、PrismaTLS 状态
- **告警** -- 配置告警阈值（证书到期、配额警告、握手失败峰值）
- **配置版本历史** -- 配置变更时间线，带差异查看器和一键恢复到任意历史版本
- **配置差异预览** -- 保存前显示已更改的字段，支持确认/取消
- **配置验证** -- 端口、地址、CIDR 格式的内联验证
- **配置导出/导入** -- 以 JSON 格式下载/上传完整服务器配置

### 配置备份

配置备份与恢复：
- **备份列表** -- 带时间戳的备份，显示名称、大小和操作
- **创建备份** -- 手动创建当前配置的快照
- **恢复** -- 从先前的备份恢复配置（恢复前自动备份当前配置）
- **差异查看器** -- 并排彩色显示备份与当前配置的差异
- **删除** -- 删除旧备份

### 流量整形

流量整形和抗分析可视化：
- **配置卡片** -- 填充模式、填充范围（字节）、时间抖动（ms）、杂音间隔、合并窗口、拥塞控制模式、目标带宽
- **Anti-RTT** -- 往返时间标准化状态和标准化值
- **桶大小分布图表** -- 柱状图（Recharts）显示填充桶大小分布
- **端口跳跃** -- 状态（启用/禁用）、基础端口、范围和间隔

## 附加功能

- **国际化** -- 完整的英文和简体中文翻译，可从侧边栏切换
- **主题切换** -- 深色、浅色和系统模式，可从侧边栏底部切换。偏好持久化存储在 localStorage 中，页面加载时自动应用。
- **通知中心** -- 持久化通知抽屉，带历史记录，可从侧边栏铃铛图标访问
- **Toast 通知系统** -- 非阻塞式 Toast 通知，用于操作反馈（成功、错误、警告、信息）。Toast 在 5 秒后自动消失，多条通知时垂直堆叠。用于配置保存确认、客户端操作、规则变更和 API 错误提示。
- **全局搜索** -- Ctrl+K 命令面板搜索页面、客户端和配置键
- **命令面板生成器** -- 十六进制（32/64/128 位）、UUID v4、base64 密钥生成器，支持一键复制到剪贴板
- **数据导出** -- 将表格导出为 CSV/JSON，图表导出为 PNG
- **客户端权限** -- 每客户端访问控制（端口转发、UDP、目标地址、端口、连接数、带宽），通过详情页配置
- **响应式侧边栏** -- 可折叠侧边栏（图标模式），移动端抽屉
- **移动端适配** -- 移动浏览器下显示底部标签导航，包含 5 个主要标签页 + 更多菜单
- **仪表盘小部件自定义** -- 在概览页重新排列和隐藏仪表盘卡片。布局偏好持久化存储在 localStorage 中。
- **多服务器管理** -- 从单个控制台实例连接多台 Prisma 服务器。侧边栏中的服务器选择器可在不同服务器之间切换，无需重新认证。
- **OpenAPI 规范** -- 自动生成的 API 文档，可通过 `/api/docs/openapi.json` 访问。可导入 Swagger UI 或其他 OpenAPI 兼容工具。
- **Prometheus 指标** -- 概览页包含抓取 URL 卡片，显示 Prometheus 兼容指标端点，可用于 Grafana/Prometheus 集成。
- **一键下载 GeoIP** -- 从服务器设置中下载并自动配置 GeoIP 数据库。服务器自动下载数据库并更新 `geoip_path` 配置。

## 开发

本地开发时，可以运行 Next.js 开发服务器：

```bash
cd apps/prisma-console
npm install
npm run dev
# → http://localhost:3000
```

开发服务器要求 Prisma 管理 API 运行在同源或已启用 CORS 的地址上。如果使用不同端口，在服务器配置中配置 `cors_origins`：

```toml
[management_api]
cors_origins = ["http://localhost:3000"]
```
